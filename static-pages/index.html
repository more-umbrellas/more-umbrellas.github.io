Static pages

12 July 2015

A static page is a page that you want to separate from your normal post content.
The most popular examples of a static page are a contact page and an about page.
These pages you want to be static so they do not show up in your post feed.

Create a static page as admin

  • create a new post
  • look for the cog wheel icon next to where it says "Save Draft" or "Update Post"
  • Click on that cog, and check the "Static Page" button.
    This will convert your post to a static page.

Info

You can define static pages with Ghost,
but you need to handle the linking to them yourself.
Static pages can be defined in page.hbs ,
which is in fact very similar to post.hbs and shares the same context.
References in Handlebars code (within the Ghost files)
appear as {{#post}}.

Notable exception to the regular posts is that
you can define different templates for different static pages.
Ghost will first look for page-{{slug}}.hbs and use it if there is one,
or fall back to page.hbs or post.hbs
The slug can be defined on the Ghost admin panel when creating a new static page.
This means that you can create as many separate pages
with different layouts that you like and
still manage all the content from the admin panel.

A static post will not show up on the index page (in the {{#foreach posts}} section)
like a regular post, the post date becomes irrelevant.
Instead it is something that has to be specifically linked to and
is used for things like About/Contact pages and in your theme
It will take on the style template from page.hbs if present or
you can make a template for a specific static post by
giving it its own template such as page-about.hbs
